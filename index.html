Overview¶

SemanticSapience-VSS is a comprehensive text analytics and NLP toolkit designed to streamline the entire workflow of:

  • Word Cloud Analysis (TF-IDF, frequency, collocation, semantic embeddings)

  • Topic Modeling (LDA, NMF, BERTopic, LSA)

  • Sentiment Analysis (rule-based, deep learning-based, plus LLM-powered ABSA and zero-shot)

  • AES-GCM-compliant project security (session key-based encryption/decryption)

  • Dynamic modals for multi-step analysis and checkpointing via a Modal History Cascade

By integrating classical NLP algorithms with modern Gen-AI approaches and robust security features, SemanticSapience-VSS addresses a variety of use cases, from academic projects to enterprise-level text analytics.

Installation¶

Clone from GitHub¶

The recommended approach is to clone the repository and install dependencies:

git clone https://github.com/vartul-shrivastava/SemanticSapience-VSS-29012025.git
cd SemanticSapience-VSS-29012025/app
pip install -r requirements.txt
python app.py

Python Version¶

  • Python 3.12+ or higher is recommended for compatibility.

Additional Dependencies¶

  • Some features (particularly LLM-based ABSA or Zero-Shot) require Ollama or additional transformer models installed locally or accessible via the Hugging Face Hub.

  • NLTK data (punkt, vader_lexicon, stopwords) is automatically downloaded on first run if not already present.

Quick Start¶

  1. Run the Flask App:

    python app.py

  2. Open the dashboard in your browser at: http://127.0.0.1:5000/

  3. Upload your CSV/XLSX dataset: - Click the Upload CSV/XLSX button and select your file. - An overview of columns and basic statistics will appear.

  4. Choose a text column for analysis: - Use the Instantiation Menu or right-click context menu for quick selection.

  5. Perform word cloud generation, topic modeling, or sentiment analysis: - Configure the respective modal parameters. - Click Run or Regenerate to visualize results in real time.

  6. Secure your project using Export Project and re-load it via Import Project at any time.

Features¶

  1. Word Cloud Analysis: - TF-IDF-based Word Cloud - Frequency-based Word Cloud - Collocation Word Cloud - Semantic Word Cloud (leveraging embeddings via SentenceTransformer)

  2. Topic Modeling: - LDA (Latent Dirichlet Allocation) - NMF (Non-negative Matrix Factorization) - BERTopic (Transformers-based topic clustering) - LSA (Latent Semantic Analysis / TruncatedSVD)

  3. Sentiment Analysis: - Rule-Based (TextBlob, VADER) - Deep Learning-based (e.g., DistilBERT from Hugging Face) - LLM-Powered:

    • Aspect-Based Sentiment Analysis (ABSA via Ollama or generative models)

    • Zero-Shot Sentiment Analysis

  4. AES-GCM Security: - Session key encryption/decryption for complete project confidentiality. - Prompt-based password entry ensures files are protected end-to-end.

  5. Dynamic Modal Architecture: - Create multiple modals to run different analyses in parallel. - Minimize or maximize modals to organize your workspace. - Use Modal History Cascades to revert or compare past checkpoint results within any modal.

How to Use¶

  1. Uploading Data: - Click Upload CSV/XLSX in the navigation bar. - After upload, columns and basic data distribution are displayed automatically.

  2. Creating a Modal: - Use the Instantiation Menu or right-click in the playground:

    • For example, LDA for topic modeling or Collocation for bigram-based word clouds.

    • Each method spawns a new draggable, resizable modal.

  3. Configuring & Running: - Inside the modal, select your text column. - Tweak parameters (e.g., maxWords, numTopics, or embeddingModel). - Click Run or Regenerate to see outputs in the preview section.

  4. Checkpointing & Comparison: - Every run can be saved as a checkpoint in the Modal History Cascade. - Restore any checkpoint to revisit older results or settings.

  5. Minimizing/Maximizing Modals: - Top-right modal buttons let you minimize to the System Tray or maximize full screen. - Access minimized modals through the System Tray panel.

  6. Export/Import Projects: - Export your current workspace to a .lspvss file with AES-GCM encryption. - Import the .lspvss file later (using the same session key or password). - Resume from exactly where you left off, including all modals, data, and checkpoints.

Tutorial¶

Basic Word Cloud¶

  1. Upload your CSV containing text reviews.

  2. Open a Frequency-based Word Cloud modal: - Select the review column. - Optionally, remove stopwords.

  3. Run: - The preview section shows your generated word cloud. - Download if desired.

Topic Modeling (LDA)¶

  1. Create an LDA modal from the menu.

  2. Choose the same text column.

  3. Set number of topics (e.g., 5) and words per topic (e.g., 5).

  4. Run: - The top words in each topic are displayed in the preview. - Export results if needed.

Sentiment Analysis¶

Rule-Based Example¶

  • Create a Rule-Based sentiment modal.

  • Select TextBlob or VADER.

  • Click Run Sentiment Analysis to view the aggregated sentiment distribution.

Deep Learning-based Example¶

  • Create a DL-Based sentiment modal.

  • Enter a Hugging Face model name (e.g., distilbert-base-uncased-finetuned-sst-2-english).

  • Run to see advanced neural sentiment labeling.

Advanced LLM Integration¶

  1. Install and verify Ollama is running if using local LLMs.

  2. Create an ABSA or Zero-Shot Sentiment modal.

  3. Select an Ollama model or any supported LLM.

  4. Run: - Results appear in the preview section with sentiment or aspect-based categorization.

API and Parameter Reference¶

Below are the main routes and the parameters they accept, based on app.py.

Word Cloud (/process/wordcloud)¶

  • method: string, one of tfidf, freq, or collocation.

  • column: string, name of the text column.

  • maxWords: integer, max number of words to display in the cloud.

  • stopwords: boolean, whether to remove English stopwords.

  • excludeWords: list of strings, additional words to exclude.

  • windowSize: integer, used only for collocation method; default is 2.

Semantic Word Cloud (/process/semantic_wordcloud)¶

  • query: string, a search or concept query.

  • column: string, text column name.

  • maxWords: integer, number of tokens to display.

  • stopwords: boolean, toggle to remove English stopwords.

  • embeddingModel: string, e.g. all-MiniLM-L6-v2.

Topic Modeling (/process/topic_modeling)¶

  • method: string, one of lda, nmf, lsa, or bertopic.

  • base64: string, base64-encoded CSV/XLSX data.

  • fileType: string, either csv or xlsx.

  • column: string, name of the text column to analyze.

  • numTopics: integer, number of topics to extract.

  • stopwords: boolean, remove English stopwords if True.

  • wordsPerTopic: integer, how many keywords per topic to display.

  • embeddingModel: string, used for bertopic; default is all-MiniLM-L6-v2.

  • randomState: integer, sets reproducible results.

Sentiment Analysis (/process/sentiment)¶

  • method: string, one of rulebasedsa or dlbasedsa.

  • base64: string, base64-encoded CSV data (only csv supported in the default route).

  • column: string, text column name.

  • ruleBasedModel: string, textblob or vader (for rule-based).

  • dlModel: string, e.g. a Hugging Face model (for deep learning-based).

Aspect-Based Sentiment Analysis (ABSA) (/process/absa)¶

  • base64: string, base64-encoded CSV/XLSX data.

  • fileType: string, csv or xlsx.

  • column: string, name of the text column.

  • aspect: string, aspect or topic of interest.

  • model: string, Ollama model name or other recognized LLM key.

Zero-Shot Sentiment Analysis (/process/zero_shot_sentiment)¶

  • base64: string, base64-encoded CSV/XLSX data.

  • fileType: string, csv or xlsx.

  • column: string, text column name.

  • model: string, Ollama model or equivalent.
